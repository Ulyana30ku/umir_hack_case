# AI Browser Assistant 🤖

Интеллектуальный ассистент для управления браузером через естественный язык. Отправляй команды голосом или текстом, и система автоматически выполнит поиск, откроет сайты и взаимодействует с веб-страницами.

## Особенности 

- **Голосовое распознавание** — говори, система понимает
- **Естественная речь** — пиши как говоришь, AI поймёт намерение
- **Автоматизация браузера** — открытие сайтов, поиск, взаимодействие с элементами
- **История сессий** — сохранение всех запросов и результатов
- **Расширение Chrome** — встроенное в браузер управление

## Установка 

### Требования
- Python 3.12+
- Node.js 18+
- Chrome/Edge браузер

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate

pip install -r requirements.txt
pip install -r hack-parser-ml/requirements.txt

python -m uvicorn main:app --reload
```

Сервер запустится на `http://127.0.0.1:8000`

### Frontend

```bash
cd front
npm install
npm run dev
```

Frontend откроется на `http://localhost:5173`

### Chrome расширение

1. Перейди на `chrome://extensions/`
2. Включи "Режим разработчика"
3. Нажми "Загрузить распакованное расширение"
4. Выбери папку `front/extension`

### Голосовой ввод

1. Кликни на кнопку  (кружечик в квадратике)
2. Говори команду
3. Система автоматически распознает речь и внедрит в текстовое поле
4. Проверь результат и нажми "Отправить"

## Как это работает 

1. **История** — система помнит контекст (какой сайт ты открывал раньше)
2. **Парсинг** — запрос обрабатывается NLP агентом:
   - Определяется намерение (открыть сайт, поиск, взаимодействие)
   - Извлекаются сущности (ключевые слова, URL)
3. **Планирование** — создаётся пошаговый план действий
4. **Выполнение** — браузер автоматизируется через Playwright
5. **Результаты** — всё логируется в истории сессии

## Интент-детекция 

Система использует **правило-based классификатор** в `QueryParser`:
- Regex-паттерны для 8 типов намерений
- 300+ маппингов сайтов
- Морфологический анализ для русского языка
- Приоритеты для конкретных доменов

Типы намерений:
- `BROWSER_NAVIGATION` — открытие сайтов
- `BROWSER_INTERACTION` — клики, скроллинг
- `PRODUCT_SEARCH` — поиск товаров
- `NEWS_SEARCH` — поиск новостей
- `INFO_RETRIEVAL` — информационный поиск
- `MULTI_STEP` — сложные многошаговые задачи
- `MIXED_TASK` — комбинация разных действий
- `UNKNOWN` — неопределённое намерение

## API Endpoints 

### Сессии
- `POST /sessions` — создать новую сессию
- `GET /sessions` — получить все сессии
- `GET /sessions/{id}/messages` — получить сообщения сессии

### Сообщения
- `POST /sessions/{id}/messages` — отправить запрос
- `GET /sessions/{id}/messages` — история сообщений

### Логи
- `GET /logs` — получить логи выполнения

## Известные ограничения 

- Некоторые сайты блокируют автоматизацию (VK, Telegram и т.д.)
- Требуется включённый микрофон для голосового ввода
- Работает в Chrome/Edge (требует расширения)
- Необходимо интернет-соединение

## Поиск неисправностей 

**Микрофон не работает:**
- Проверь доступ браузера к микрофону
- Убедись, что включено распознавание речи в браузере

**Сайты не открываются:**
- Перезагрузи расширение Chrome
- Проверь console браузера на ошибки

**Запросы не обрабатываются:**
- Проверь, запущены ли backend и ML агент
- Посмотри логи в `/backend/hack-parser-ml`
